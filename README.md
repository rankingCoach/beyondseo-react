# BeyondSEO React App

This is the React front-end for the **BeyondSEO WordPress plugin**, built with [Vite](https://vitejs.dev/) and [React 19](https://react.dev/).

The compiled output (`dist/`) is loaded by the PHP plugin at runtime. This repository must be cloned directly into the `react/` folder inside the BeyondSEO plugin:

```bash
cd wp-content/plugins/beyondseo
git clone https://github.com/rankingCoach/beyondseo-react react
```

This ensures the project lands at `wp-content/plugins/beyondseo/react/`, which is the path the PHP plugin expects.

---

## Requirements

| Tool | Minimum version |
|------|----------------|
| Node.js | 20.x or higher |
| npm | bundled with Node 20 |
| jq *(macOS, for SDK generation only)* | any recent version |

> **Tip (macOS):** Install Node via [nvm](https://github.com/nvm-sh/nvm) or [Homebrew](https://brew.sh). Install `jq` with `brew install jq`.

---

## Installation

Navigate into the `react/` folder and install dependencies:

```bash
cd wp-content/plugins/beyondseo/react
npm install
```

---

## Build

Compile the application for production. The output is written to `react/dist/` and is loaded by the PHP plugin automatically.

```bash
npm run build
```

---

## Updating the SDK (when PHP changes)

Whenever the **BeyondSEO PHP plugin API** is modified (e.g. new endpoints, changed response shapes), the TypeScript SDK types must be regenerated before rebuilding.

### Step 1 — Get a token

Run the token helper. It will prompt you for your local WordPress host, username, and an [application password](https://developer.wordpress.org/rest-api/using-the-rest-api/authentication/#application-passwords):

```bash
npm run gen:TOKEN
```

The script prints the exact command to run next, for example:

```
npm run gen:SDK -- --base=http://localhost:8080 --token=<YOUR_TOKEN>
```

### Step 2 — Generate the SDK

Copy the command printed by the previous step and run it:

```bash
npm run gen:SDK -- --base=<YOUR_HOST> --token=<YOUR_TOKEN>
```

> If your host uses **HTTPS with a self-signed certificate** (e.g. a local dev environment), prefix the command with `NODE_TLS_REJECT_UNAUTHORIZED=0`:
>
> ```bash
> NODE_TLS_REJECT_UNAUTHORIZED=0 npm run gen:SDK -- --base=https://localhost:8080 --token=<YOUR_TOKEN>
> ```

### Step 3 — Rebuild

After the SDK is regenerated, build the application again:

```bash
npm run build
```

## Full workflow summary

```bash
# 1. Install dependencies (first time or after package.json changes)
npm install

# 2. Build the app
npm run build

# ---- Only needed when the PHP plugin API has changed ----

# 3. Get a fresh token (follow the prompts)
npm run gen:TOKEN

# 4. Generate the SDK types (use the command printed by step 3)
npm run gen:SDK -- --base=<YOUR_HOST> --token=<YOUR_TOKEN>

# 5. Rebuild
npm run build
```

## Project structure

```
react/
├── dist/               # Compiled output (loaded by the PHP plugin)
├── generators/         # SDK & token generation scripts
│   └── get-sdk-token.sh
├── src/
│   ├── components/     # React components
│   ├── models/         # Auto-generated SDK models (do not edit manually)
│   ├── stores/         # Redux stores
│   ├── helpers/
│   ├── custom-hooks/
│   ├── contexts/
│   ├── assets/
│   ├── App.tsx
│   └── main.tsx
├── package.json
└── vite.config.js
```

> **Important:** Do **not** manually edit files inside `src/models/` or `src/stores/` that are auto-generated by the SDK generator. Always re-run `npm run gen:SDK` to update them.

## Available scripts

| Command | Description |
|---------|-------------|
| `npm install` | Install all dependencies |
| `npm run build` | Production build → `dist/` |
| `npm run dev` | Start local Vite dev server |
| `npm run watch` | Watch mode — rebuilds on file changes |
| `npm run gen:TOKEN` | Interactive helper to fetch an SDK token |
| `npm run gen:SDK` | Regenerate TypeScript SDK types from the PHP API |
| `npm run lint` | Run all linters (Prettier, Stylelint, ESLint) |
| `npm run format` | Auto-fix all lint issues |
| `npm run clean` | Remove `node_modules` and reinstall from scratch |
